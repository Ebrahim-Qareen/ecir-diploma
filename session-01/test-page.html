A SIEM is the platform that collects logs from everywhere, connects them together, and tells you when something looks wrong.

Why it exists

Every system in an organization writes logs, but they live in a hundred different places. Nobody can watch a hundred places at once. The SIEM brings them into one screen so a single analyst can defend the whole company.

How it works

  • An agent on each machine collects logs and applies parsing configuration.
  • Logs travel over the network to the manager (the SIEM server).
  • The manager stores, indexes, and runs detection rules against them.
  • When a rule matches, an alert lands in the analyst's queue.

Callout boxes

SOC Perspective

As a Tier 1 analyst, the SIEM is your main screen. Everything you see there exists because somebody configured collection correctly. A missing log source is a blind spot you will never notice — until an attacker uses it.

Common Mistake

Thinking a SIEM equals security. Companies buy an expensive platform, forward nothing useful into it, and believe they are protected. Tools without telemetry are storage, not defence.

Exam Tip

Remember: SIEM = SIM + SEM. SIM is historical log management, SEM is real-time monitoring and alerting.

Commands and code

Inline commands look like index=wineventlog EventCode=4625, and blocks look like this:

# Enable the receiver on the SIEM server
sudo /opt/splunk/bin/splunk enable listen 9997 -auth admin:password

# Point a forwarder at it
sudo /opt/splunkforwarder/bin/splunk add forward-server 10.10.30.10:9997

Lab steps

  1. Install Splunk on SOC-CORE01.

    Download the .deb package, then install and start it.

    sudo dpkg -i splunk-*.deb
sudo /opt/splunk/bin/splunk start --accept-license
  2. Create your indexes.

    Separate indexes keep Windows and Linux data clean and searchable.

  3. Install the forwarder on the victim.

    This is the agent we discussed in Session 1 — now it becomes real.

✅ Verification

  • Splunk web interface opens at http://10.10.30.10:8000
  • index=wineventlog returns events
  • Logging out and back in on the victim produces a new 4624 within one minute
  • Snapshot taken: L2-complete

Knowledge check

Q1 Where does log parsing configuration live — on the agent or the manager?

Show answer

On the agent. The agent decides what to collect and how to read it before shipping. The manager holds storage and the detection rules.

Q2 An alert fires but the activity turns out to be an administrator's legitimate script. What is this called, and why does it matter?

Show answer

A false positive. Too many of them cause alert fatigue, which is how real attacks get closed by mistake.

Key takeaways

  • A SIEM collects logs from everywhere and puts them on one screen.
  • The agent collects and parses; the manager stores and runs the rules.
  • SIEM = SIM (history) + SEM (real time).
  • No logs means no detection — collection is where security starts.
