Host app-prod-02 triggered alerts overnight. Reconstruct the intrusion. Index endpoint_logs. Fields: timestamp, host, event_type, src_ip, src_country, user, process, command, result, dest_ip, dest_port, file_name, file_hash.

  • backupsvc

  • deploy

Q3 Which tool + C2 endpoint pulled the second-stage payload?

  • scp from 45.9.148.37:22

  • wget from 185.220.101.45:8080

Q6 Which TWO persistence mechanisms were established?

  • Cron /etc/cron.d/mysql-backup + UID-0 user sysupd

  • Registry Run key + scheduled task

Q7 SHA-256 of the dropper (update.sh) for blocklisting?

  • 1a7f3e9c2b8d…3a2 (miner)

Kill chain (MITRE)

T1110 Brute Force → T1078 Valid Accounts → T1105 Ingress Tool Transfer → T1496 Resource Hijacking → T1053.003 Cron / T1136.001 Local Account. Dwell ≈ 11–12 min.

Instructor · SPL solutions 
# Q1 brute-force source
index=endpoint_logs event_type=authentication result=failed | stats count by src_ip | sort - count
# Q2 first success from attacker IP
index=endpoint_logs event_type=authentication result=success src_ip="45.9.148.37" | sort + timestamp | table timestamp user
# Q3 download tool + C2
index=endpoint_logs event_type=process (command="*wget*" OR command="*curl*") | table timestamp user command
# Q4 + Q7 dropper name + hash
index=endpoint_logs event_type=file | table timestamp process file_name file_hash
# Q5 miner beacon (network, not command line)
index=endpoint_logs event_type=network dest_port=8080 | stats count by dest_ip dest_port
# Q6 persistence
index=endpoint_logs (process=useradd OR file_name="mysql-backup" OR command="*cron*") | table timestamp user process command file_name

Scenario 2 · Web Application Attack Chain

shop.cybert-lab.com was targeted on 2024-04-11. Trace the full kill chain in the access logs. Index web_logs. Fields: timestamp, src_ip, src_country, method, uri, status, bytes, user_agent, referer, http_version, host.

  • 193.106.191.22

  • 151.1.37.69

Q2 Which tool did the initial enumeration?

  • sqlmap/1.8.2
  • Nikto/2.5
  • gobuster/3.6
  • curl/8.2

Q3 Which endpoint was brute-forced, and which status marked success?

Q6 Web-shell filename and first command executed?

  • cmd.php — id
  • shell.php — whoami

Q7 The secondary payload was fetched from which IP, to what path?

  • 185.220.101.45 → /tmp/x
  • 45.9.148.37 → /var/www/bd
  • 193.106.191.22 → /tmp/bd

OWASP mapping (bonus)

Dir enum → recon · brute force → A07 Auth Failures · SQLi → A03 Injection · path traversal → A01 Broken Access Control · web shell + RCE → A03 Injection.

Instructor · SPL solutions 
# Q1 dir brute-force source
index=web_logs status=404 | stats count by src_ip | sort - count
# Q2 tool (filter to the 404 flood first)
index=web_logs src_ip="193.106.191.22" status=404 | stats count by user_agent | sort - count
# Q3 login brute force + success (401 fail, 302 success)
index=web_logs uri="/admin/login" | stats count by status
# Q4 SQLi payloads
index=web_logs uri="*UNION*" OR uri="*'%20OR%20'1'='1*" | table timestamp uri status
# Q5 path traversal outcomes
index=web_logs uri="/download*" | table timestamp uri status
# Q6 web shell + commands
index=web_logs uri="*shell.php*" | sort + timestamp | table timestamp uri status
# Q7 secondary payload (URL-decode cmd)
index=web_logs uri="*shell.php*wget*" | table timestamp uri
How to run this in class

Give students the dataset + questions only. They ingest, investigate, and answer. Reveal the SPL and the answer feedback for grading. These three challenges map to Sessions 2–4 (SIEM & detection); harder DFIR challenges arrive in Sessions 9–10.
